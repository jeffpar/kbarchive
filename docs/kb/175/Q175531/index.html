Q175531: White Paper: Migrating Applications to Visual Basic 5.0

Article: Q175531
Product(s): Microsoft Visual Basic for Windows
Version(s): WINDOWS:5.0
Operating System(s): 
Keyword(s): kbfile kbVBp500
Last Modified: 11-JAN-2001

-------------------------------------------------------------------------------
The information in this article applies to:

- Microsoft Visual Basic Learning Edition for Windows, version 5.0 
- Microsoft Visual Basic Professional Edition for Windows, version 5.0 
- Microsoft Visual Basic Enterprise Edition for Windows, version 5.0 
-------------------------------------------------------------------------------

SUMMARY
=======

Visual Basic 5.0 offers many new features and functions that make it a
worthwhile platform to move to. Of course, the most compelling reason may be the
fact that it now creates native code executables, resulting in performance gains
over previous versions of Visual Basic. In addition, you have the ability to
create ActiveX controls, ActiveDocs, as well as other features.

MORE INFORMATION
================

This document considers and discusses the following:

- Reasons to migrate

- Issues to Consider

- Notable Changes to Visual Basic

- Types of Change

- Plan of Attack

- Migration Resources

The following file is available for download from the Microsoft Download Center:

  Migrate.exe
  (http://download.microsoft.com/download/vb50ent/Sample5/1/W9XNT4/EN-US/Migrate.exe)

For additional information about how to download Microsoft Support files, click
the article number below to view the article in the Microsoft Knowledge Base:

  Q119591 How to Obtain Microsoft Support Files from Online Services

posting to scan this file for viruses. Once posted, the file is housed on secure
