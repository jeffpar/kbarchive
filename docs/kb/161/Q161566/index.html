Q161566: GDI Heap Corruption Causes Stop 0x0000000A

Article: Q161566
Product(s): Microsoft Windows NT
Version(s): 3.51
Operating System(s): 
Keyword(s): kbnetwork
Last Modified: 09-AUG-2001

-------------------------------------------------------------------------------
The information in this article applies to:

- Microsoft Windows NT Server version 3.51 
- Microsoft Windows NT Workstation version 3.51 
-------------------------------------------------------------------------------

SYMPTOMS
========

In Microsoft Windows NT 3.51 Server or Workstation, running a WIN32 GDI heap
corruption may cause a Stop 0x0000000A bugcheck.

STATUS
======

Microsoft has confirmed this to be a problem in Microsoft Windows NT 3.51 Server
and Workstation. This problem has been corrected in the latest U.S. Service Pack
for Microsoft Windows NT 3.51 Server and Workstation. For information on
obtaining the Service Pack, query on the following word in the Microsoft
Knowledge Base (without the spaces):

  S E R V P A C K
