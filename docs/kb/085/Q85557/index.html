Q85557: File Manager Shows Garbled Date for Year 2000 or Later

Article: Q85557
Product(s): Microsoft Windows 3.x Retail Product
Version(s): 3.1,3.11
Operating System(s): 
Keyword(s): kbfile kbtool kbYear2000 win31 kbgraphxlinkcritical
Last Modified: 13-AUG-2001

-------------------------------------------------------------------------------
The information in this article applies to:

- Microsoft Windows versions 3.1, 3.11 
- Microsoft Windows for Workgroups versions 3.1, 3.11 
-------------------------------------------------------------------------------

SYMPTOMS
========

Microsoft Windows File Manager displays an incorrect date if the file is created
with a date of 01-01-2000 or later.

STATUS
======

Microsoft has confirmed this to be a problem in the Microsoft products that are
listed at the beginning of this article.

MORE INFORMATION
================

NOTE: You do not need to install this version of Winfile.exe if you have already
applied W31filup.exe or Wfwfilup.exe.

With the original Winfile.exe, any date beyond 2000 is displayed as (11/10/:0).
The date after the File Manager update appears like this (11/10/00).

For information about a similar issue with Microsoft Windows 95, see the
following article in the Microsoft Knowledge Base:

  Q172707 File Manager Shows Incorrect Date For Year 2000 or Later
