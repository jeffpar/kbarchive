Q192239: Postmaster Messages Display Incorrect Date After 2000

Article: Q192239
Product(s): Microsoft Mail For PC Networks
Version(s): WINDOWS:2.x,3.0,3.2,3.5
Operating System(s): 
Keyword(s): kbYear2000
Last Modified: 19-JAN-2000

-------------------------------------------------------------------------------
The information in this article applies to:

- Microsoft Mail for Windows, versions 2.x, 3.0, 3.2, 3.5 
-------------------------------------------------------------------------------

SYMPTOMS
========

When the Postmaster sends a message to a user after the year 2000, the message
displays 10 or 100 instead of the correct date. Delivery receipts and read
receipts may also show the same incorrect date.

CAUSE
=====

An older version of MS Mail is being used.

RESOLUTION
==========

To resolve this problem, contact Microsoft Product Support Services to obtain
the following fix.

This fix should have the following time stamp:

  Async.ovl      6/26/97   11:34a   266,962 bytes (version 3.5.34)
  External.exe   6/26/97   11:34a    53,408 bytes (version 3.5.34)
  X25eicon.ovl   6/26/97   11:35a   264,290 bytes (version 3.5.34)

This hotfix has been posted to the following Internet location as Exty2k.exe:

  ftp://ftp.microsoft.com/bussys/mail/pcmail-public/All-Y2K/

STATUS
======

Microsoft has confirmed this to be a problem in the versions of Microsoft Mail
for Windows listed at the beginning of this article. A supported fix is now
available, but has not been fully regression-tested and should be applied only
to systems experiencing this specific problem. Contact Microsoft Product Support
Services for more information.
