Q240049: How to Obtain Diagnostic i386 Processsor Integrity Tests

Article: Q240049
Product(s): Microsoft Windows NT
Version(s): winnt:4.0
Operating System(s): 
Keyword(s): kbgraphxlinkcritical
Last Modified: 10-AUG-2001

-------------------------------------------------------------------------------
The information in this article applies to:

- Microsoft Windows NT Server version 4.0 
- Microsoft Windows NT Server, Enterprise Edition version 4.0 
-------------------------------------------------------------------------------

SUMMARY
=======

The Trusted Facility Manual (TFM) for TCSEC C2 certification for Windows NT 4.0
Service Pack 6 requires that a suite of diagnostic x86 processor integrity tests
be made available to licensees of Windows NT 4.0.

As stated in Section 2.2.3.1.2 of the Department of Defense Trusted Computer
Systems Evaluation Criteria (DOD 5200.28-STD):

  System Integrity

  Hardware and/or software features shall be provided that can be used to
  periodically validate the correct operation of the on-site hardware and
  firmware elements of the TCB.

The Microsoft Hardware System Integrity Test Tool (HSITT) can be download from
the Microsoft Download Center. Click the file name below to download the file:

  DownloadDownload Hsitt.exe now
  (http://www.microsoft.com/downloads/release.asp?releaseid=20566)

The Hardware System Integrity Test Tool requires Windows NT 4.0 with Service Pack
6a and the post-SP6a "C2 Update", as discussed at
http://www.microsoft.com/technet/security/C2config.asp.

For additional information about how to download Microsoft Support files, click
the article number below to view the article in the Microsoft Knowledge Base:

  Q119591 How to Obtain Microsoft Support Files from Online Services

Microsoft used the most current virus detection software available on the date of
posting to scan this file for viruses. Once posted, the file is housed on secure
servers that prevent any unauthorized changes to the file.
