Q195751: Encarta: Selecting Special Character Disables Read Aloud Feature

Article: Q195751
Product(s): Microsoft Home Multimedia Titles
Version(s): WINDOWS:
Operating System(s): 
Keyword(s): kbtool kbimu
Last Modified: 19-DEC-1999

-------------------------------------------------------------------------------
The information in this article applies to:

- Microsoft Encarta Encyclopedia 99 
-------------------------------------------------------------------------------

SYMPTOMS
========

If you select a special character in Encarta Encyclopedia 99 and attempt to use
the Read Aloud feature to pronounce the name of the character, the Read Aloud
feature may no longer function.

NOTE: Depending on which character you select, the Read Aloud feature may or may
not pronounce the name of the character.

RESOLUTION
==========

To resolve this issue, follow these steps:

1. Click Options, point to Read Aloud, and then click Settings.

2. Click Test.

3. After the test speech is read aloud, click OK.

STATUS
======

Microsoft has confirmed this to be a problem in Microsoft Encarta Encyclopedia
99.

MORE INFORMATION
================

This issue has been reported with the following special characters:

- Asterisk

- Hyphen

- Underscore

NOTE: This issue occurs only if you select the special character without any
accompanying text.
